## 網頁功能

### 分頁一：持股清單
- 上方下拉選單切換不同 ETF
- 顯示欄位：序號、股票代號+名稱、股數、股價、持股比例、狀態、加/減碼股數、加/減碼金額
- **狀態欄**：新增（紫）、出清（橘）、加碼（紅）、減碼（綠）
- **比例欄**：顯示前一日 → 今日的箭頭變化
- **加/減碼金額欄**：可點擊排序（由大到小 / 由小到大 / 預設）
- Header 顯示：經理人、最新股價、今年以來 YTD 績效

### 分頁二：ETF持股交叉比對
- 同時載入所有 ETF 資料
- 僅顯示**被 2 檔以上 ETF 同時持有**的股票
- 每筆顯示：股票代號+名稱、持有 ETF 清單（含各 ETF 持股比例）、持有 ETF 數量
- 可點擊「持有 ETF 數」欄位排序

## 每日自動更新流程

### 00981A（統一台股增長）
1. 從 ezmoney.com.tw 用 Playwright 下載持股 XLSX
2. 讀取 Excel 標頭的民國日期（例：`115/04/17`），確認是否為今日資料
3. 若日期符合今日：解析持股、與前一日比對、取得股價（yfinance）、產生 `data_00981A.json`、推送 GitHub、發送 Telegram 通知
4. 若日期不符（尚未更新）：結束，等下一個小時排程再試

### 00992A / 00982A（群益系列）
1. 從 capitalfund.com.tw 用 Playwright 下載持股 XLSX
2. **特殊日期邏輯**：網站選擇日期 D 會回傳前一個交易日的資料
   - 腳本執行當天（T+1）→ 在網站選擇 T+1 → 取得 T 的資料
   - 使用 JavaScript 設定 Angular 日期選擇器（直接 `.type()` 無效）
3. 解析 Excel 第二個 sheet（參股）
4. 與前一日持股比對、取得股價、產生 JSON、推送 GitHub、發送 Telegram 通知

## Windows 排程設定

目前設定為每個交易日（週一～週五）下午 4 點起，每小時執行一次，持續到晚上 10:30。

| 重複間隔 | 每 1 小時 |
| 持續時間 | 6 小時 30 分 |
| 動作 | `d:\Self_Tools\ETF_Tracker\venv\Scripts\python.exe` |
| 引數 | `d:\Self_Tools\ETF_Tracker\check_and_update_00992A.py`（或 00982A） |

## Telegram 通知設定

通知憑證存放在 `.env` 檔案（不會 commit 到 GitHub）：

```
# d:\Self_Tools\ETF_Tracker\.env
TELEGRAM_BOT_TOKEN=你的BotToken
TELEGRAM_CHAT_ID=你的ChatID
```

### 取得 Chat ID
1. 在 Telegram 傳訊息給你的 Bot
2. 瀏覽 `https://api.telegram.org/bot{TOKEN}/getUpdates`
3. 找 `"chat": {"id": ...}` 的數字

### 通知內容
每日更新完成後自動發送，包含：
- ETF 代號、資料日期、股價、YTD 績效
- 加碼/減碼/新增/出清 各幾檔
- 新增、出清的個股名稱與張數
- 加碼、減碼的個股名稱與張數（依變動量排序）

## 新增 ETF 步驟

1. **確認資料來源**
   - 統一投信系列 → 參考 `check_and_update.py`（ezmoney.com.tw）
   - 群益投信系列 → 參考 `check_and_update_00992A.py`（capitalfund.com.tw），找到該 ETF 的 product ID（URL 中的數字）

2. **複製腳本並修改 Config**
   ```python
   FUND_URL = "https://www.capitalfund.com.tw/etf/product/detail/{產品ID}/portfolio"
   DATA_FILE = "data_{代號}.json"
   ETF_CODE = "{代號}"
   MANAGER = "{經理人姓名}"
   ```
   同時更新 log 檔名、git commit 訊息、`build_notification()` 的 etf_code / etf_name 參數。

3. **下載初始歷史資料**（至少兩天，才能產生差異比對）
   參考 `_init_00982A.py` 的做法撰寫一次性下載腳本。

4. **更新網頁**
   - `index.html`：在 `<select id="etf-selector">` 新增 `<option>`
   - `app.js`：在 `ETF_LIST` 陣列新增 `{ id, name }`

5. **設定 Windows 排程**（參考上方排程設定表格）

6. **Git push**，GitHub Pages 約 1-5 分鐘後生效

## 已知注意事項

- **群益網站日期邏輯**：選擇日期 D 回傳的是前一個交易日資料，腳本已處理此偏移。
- **Angular 日期選擇器**：必須用 JavaScript `HTMLInputElement.prototype.value` setter 設定，直接用 Playwright 的 `.type()` 會填錯欄位。
- **假日不處理**：跳過週末，但國定假日未處理（網站若無資料，腳本會在下一次排程重試）。
- **yfinance 部分股票查無價格**：上市/上櫃代號有時需要嘗試 `.TW` 和 `.TWO` 兩種後綴，腳本已自動嘗試兩者，查無時股價顯示 0。
- **`.env` 不可 commit**：已在 `.gitignore` 排除，換機器時需重新建立。
- **data_*.json 為前端直接讀取的資料**：每次更新後必須 git push 才會反映到網站。
