## 網頁功能

### 1. 今年以來績效排名（YTD）
- 位於 Header 下方、分頁按鈕上方
- 同時讀取所有 9 檔 ETF 的 data JSON，依 YTD 由高到低排序
- 前三名顯示金/銀/銅漸層圓形獎牌（含彩色光暈），其餘顯示灰色圓形序號
- 點擊任一列可直接切換至該 ETF 的持股清單，同步高亮顯示選中項
- 標題下方顯示「加權指數績效 +x.xx%」（紅色正值、綠色負值）

### 2. 持股清單（Holdings Tab）
- ETF 選擇器（下拉選單）切換標的
- 顯示欄位：序號、股票代號/名稱、股價（含日期）、股數（前後對比格式）、持股比例（含前後比較箭頭）、狀態（新增/出清/加碼/減碼）、加減碼股數、加減碼金額
- **欄位說明**
  - **股價**：最新股價，後方以灰色小字顯示資料日期（如 `12.34 (2026-04-22)`）
  - **股數**：若有變動，以 `灰色舊值 → 彩色新值` 格式顯示；無變動僅顯示當前值
  - **比例**：同股數格式，`灰色舊值 → 彩色新值`
- **排序功能**：
  - 加減碼金額：點擊欄位標題，循環切換 降冪 → 升冪 → 預設（按 todayWeight）
  - 加減碼股數：同上
  - 狀態：按 新增(4) > 加碼(3) > 持平(2) > 減碼(1) > 出清(0) 優先序排序
- Header 顯示：ETF 名稱、經理人、ETF 當日股價（含更新日期）、YTD 績效
- 顯示資料日期（最新交易日）及最後更新時間

### 3. ETF 持股交叉比對（Cross Tab）
- 彙整所有 9 檔 ETF 持股，找出被 **2 檔以上** ETF 同時持有的標的
- 顯示：序號、股票代號/名稱、持有的 ETF 列表（含各自持股比例）、持有 ETF 數量
- 持有 ETF 數欄可點擊排序（降冪/升冪）
- 切換至此 Tab 時，Header 的「基金規模/市值」資訊自動隱藏

## 自動化機制（GitHub Actions）

### Workflow 1：daily_update.yml（持股更新）

**觸發時間（台灣時間，僅平日）**

| UTC | 台灣時間 | 說明 |
|-----|---------|------|
| 08:00 | 16:00 | 第一次嘗試 |
| 09:00 | 17:00 | 若未更新則重試 |
| 10:00 | 18:00 | 最後重試 |

**執行流程：**
1. checkout 最新 repo
2. 安裝 Python 3.11 + pip 套件 + Playwright Chromium
3. 依序執行 9 支腳本（每支腳本自行判斷是否需要更新）
4. `git add data_*.json holdings/` → commit → push

- 若持股已更新：腳本直接跳過，不重複通知
- 若持股尚未更新：發送 Telegram 等待訊息，下一個整點再試
- **注意**：此 workflow 的 commit 只更新 `lastUpdate`（持股更新時間）

### Workflow 2：price_update.yml（股價更新）

每日收盤後（台灣時間約下午 3–4 點）執行 `update_prices.py`：
- 更新所有 9 檔 ETF 的 `etfPrice`、`ytd`、`priceDate`（不更動持股資料）
- 更新 `data_index.json`（加權指數 ^TWII YTD）
- **注意**：此 workflow **不更動** `lastUpdate`，避免與持股更新時間混淆

## 各腳本更新邏輯

### 通用流程（所有腳本）
1. 計算前一交易日（跳過週末）
2. 若 `holdings/` 已存在該日快照 → **直接結束**（不重複抓取）
3. 向來源網站抓取今日持股
4. 若持股資料為空 → 發送 Telegram 等待通知後結束（等下次重試）
5. 儲存持股快照至 `holdings/{CODE}_holdings_{date}.json`
6. 讀取最近一筆歷史快照作為比較基準
7. 用 yfinance 取得每檔股票當日收盤價（先試 `.TW`，失敗再試 `.TWO`）
8. 計算差異（股數、金額）、YTD、ETF 股價
9. 產生 `data_{CODE}.json`
10. `git push`（觸發 GitHub Pages 更新）
11. 發送 Telegram 成功通知（含加碼/減碼/新增/出清明細）

### 各腳本資料來源說明

#### 類型 A：ezmoney.com.tw（00981A）
- Playwright 開啟網頁，點擊下載 XLSX
- 驗證 XLSX 民國日期欄位是否符合前一交易日
- 從第 19 行起解析持股

#### 類型 B：Nomura JSON API（00980A、00985A）
- `POST https://www.nomurafunds.com.tw/API/ETFAPI/api/Fund/GetFundAssets`
- Payload：`{"FundID": "00980A", "SearchDate": "YYYY-MM-DD"}`
- 純 HTTP 請求，無需 Playwright

#### 類型 C：fhtrust.com.tw（00991A）
- `GET https://www.fhtrust.com.tw/api/assetsExcel/ETF23/{YYYYMMDD}`
- 直接下載 XLSX，無需 Playwright

#### 類型 D：capitalfund.com.tw（00992A、00982A）
- Playwright 開啟 Angular 頁面
- 必須用 JS `HTMLInputElement.prototype.value` setter + `dispatchEvent` 設定日期（Angular 雙向綁定，不能用 Playwright `.type()`）
- **日期偏移**：填入 run_date（當天），實際取得前一交易日資料（已在腳本中處理）
- 產品 ID：00992A = 500，00982A = 399

#### 類型 E：tsit.com.tw（00987A）
- `GET https://www.tsit.com.tw/ETF/Home/ETFSeriesDetail/00987A`
- 純靜態 HTML，urllib 直接抓取，無需 Playwright
- 持股表格中股票代碼含 ` TT` 後綴（如 `2330 TT`），解析時自動去除
- 用 `re.fullmatch(r'\d{4}', code)` 辨識有效股票代碼列

#### 類型 F：etf.allianzgi.com.tw（00993A）
- 目標：`https://etf.allianzgi.com.tw/etf-info/E0002?tab=4`
- Playwright 開啟頁面，**攔截 XHR 回應**（`page.on("response", handler)`）
- 先瀏覽 `tab=1` 取得 `GetFundDetail`（含經理人姓名），再瀏覽 `tab=4` 取得 `GetFundAssets`（含持股）
- Allianz API 有 CSRF 保護，直接 HTTP 請求會得到 400；必須在 Playwright 瀏覽器 session 內攔截
- 解析：`tables[1]["Rows"]` = 持股列表，`tables[2]["Rows"]` = 基金規模
- 資料日期：`PCFDate` 欄位（Allianz 慣例：PCFDate 為次一交易日，`dataDate` 直接使用此值）

#### 類型 G：ctbcinvestments.com.tw（00995A）
- 無需 Playwright，純 HTTP 三步驟：
  1. `POST /API/home/AuthToken` → 取得 `token`
  2. `POST /API/etf/ETFDetail?token=...` → 取得經理人姓名
  3. `POST /API/etf/ETFHoldingWeight?token=...` → 取得持股資料（`FID: "E0036"`）
- Bootstrap 邏輯：若無歷史快照，自動從 CTBC 補抓前一交易日資料
- AUM/股數：從 `FundAssets[0]` 取最大兩個數字格式字串（逗號分隔整數）

## data_{CODE}.json 格式

```json
{
  "meta": {
    "manager": "魏永祥",
    "ytd": "5.23",
    "etfPrice": 12.34,
    "priceDate": "2026-04-22",
    "dataDate": "2026-04-21",
    "lastUpdate": "2026-04-21 16:15",
    "totalShares": 12345,
    "prevTotalShares": 12300,
    "totalMarketCap": 15.23,
    "prevTotalMarketCap": 15.18
  },
  "holdings": [
    {
      "rank": 1,
      "code": "2330",
      "name": "台積電",
      "shares": 1000000,
      "prevShares": 980000,
      "price": 950.00,
      "yestWeight": 28.5,
      "todayWeight": 29.1,
      "diffShares": 20000,
      "diffAmount": 19000000.0
    }
  ]
}
```

**欄位說明（新增欄位）：**

| 欄位 | 說明 |
|------|------|
| `priceDate` | 股價更新日期（由 `update_prices.py` / `price_update.yml` 寫入） |
| `lastUpdate` | 持股資料最後更新時間（僅由各 `check_and_update_*.py` 寫入） |
| `prevTotalShares` | 上一次持股更新時的基金規模（張數） |
| `prevTotalMarketCap` | 上一次持股更新時的總市值（億元） |

```
TELEGRAM_BOT_TOKEN=你的BotToken
TELEGRAM_CHAT_ID=你的ChatID
```

### GitHub Actions
至 GitHub 倉庫 → **Settings → Secrets and variables → Actions → Repository secrets** 新增：

| Secret 名稱 | 說明 |
|------------|------|
| `TELEGRAM_BOT_TOKEN` | Telegram Bot API Token |
| `TELEGRAM_CHAT_ID` | 接收通知的 Chat ID |

> **注意**：必須設在 Repository secrets，不是 Environment secrets（Workflow 未指定 environment，Environment secrets 無法讀取）。

## 新增 ETF 步驟

1. **確認資料來源類型**（A/B/C/D/E/F/G，見上方說明），選最相近的腳本複製
2. **複製腳本**：`check_and_update_{新代號}.py`
3. 修改腳本頂部常數：`ETF_CODE`、`MANAGER`、API URL 或 Product ID
4. **index.html**：在 `<select id="etf-selector">` 新增 `<option value="{代號}">`
5. **app.js**：在 `ALL_ETFS` 和 `ETF_LIST` 兩個陣列中各新增 `{ id: '代號', name: '名稱' }`
6. **update_prices.py**：在 `ETFS` 清單中新增 `("{代號}", "data_{代號}.json")`
7. **daily_update.yml**：在 `Run all ETF update scripts` 步驟新增一行 `python check_and_update_{代號}.py`
8. 執行腳本產生初始 `data_{CODE}.json`
9. Git push，GitHub Pages 約 1–5 分鐘後生效

## 常見問題

| 問題 | 原因 | 解法 |
|------|------|------|
| Angular 日期選擇器填值無效 | `.type()` 不觸發 Angular 雙向綁定 | 用 JS prototype setter + `dispatchEvent` |
| capitalfund 資料日期偏移一天 | 網站選 T 日返回 T-1 資料 | 腳本已處理，勿調整日期邏輯 |
| Telegram 400 Bad Request | 股票名稱含 HTML 特殊字元 | 勿使用 `parse_mode=HTML` |
| yfinance 股價為 0.0 | 未上市、休市或盤中（尚未收盤）| 正常現象，收盤後自動補正；依序試 `.TW`、`.TWO` |
| GitHub Pages 未更新 | 未 push 或傳播延遲 | 等待 1–5 分鐘；確認 Actions 執行成功 |
| Telegram 通知未收到 | Secrets 設在 Environment 而非 Repository | 移至 Repository secrets（Settings → Secrets → Actions）|
| GitHub Actions Node.js 警告 | actions 尚未原生支援 Node 24 | Workflow 已設 `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: true` |
| tsit.com.tw 解析到 0 筆持股 | HTML 結構異動或解析器過早停止 | 確認 `HoldingsTableParser` 掃描全部 `<tr>`，勿限制表格編號 |
| Allianz GetFundAssets 回傳 400 | CSRF 保護，直接 HTTP 請求被拒絕 | 必須在 Playwright session 內攔截 XHR，不可改用純 HTTP |
| CTBC Playwright headless 被擋 | WAF 偵測到 headless 瀏覽器 | 改用純 HTTP + AuthToken 流程（無需 Playwright）|
| lastUpdate 顯示股價更新時間 | update_prices.py 誤更動 lastUpdate | 確認 update_prices.py 只寫 priceDate，不寫 lastUpdate |
| 網頁股價日期與持股日期不同 | 兩者由不同 workflow 更新 | 正常：priceDate 由 price_update.yml 更新，dataDate 由 daily_update.yml 更新 |
